DESCRIPTION OF APPLICATION

The project is the development of a calculator that provides a user with settings required for CNC milling of a variety of materials. Currently the application only provides data for 2D milling where the bit is always engaged with material on both edges. This will be expanded on in future development.

The calculator uses a script that loads a modal via startButton click. From there the user is asked to select from an array materials. Once the materials has been selected by the user a selectMaterial.addEventListener triggers the thickness.style.visibility = "visible". Once the user enters a thickness of the material to be calculated the rpm.style.visibility = "visible" is triggered and the user is prompted to enter the RPM (rotations per minute). The range is between 5000 and 60000. If user enters anything below or above they will receive a window.alert prompting them to enter a value between this. Same applies for the material thickness if the user enters anything above 50mm.

Once the parameters have been entered by the user a bitsButton.onclick fires a function that closes the current modal and opens another. The application then stores the values entered in to an array and displays this to the user. A const Material is then created as well as bits which is then filtered by bits.plastic, bits.metal, and bits.multipurpose comparing them to the material array comparing the boolean for true for these values. The filter also checks that the user input thicknessValue is less than the bits.cel. This then populates the select options to display for the user to select from and the

When the user has chosen a bit the selectBitButton.onclick function closes the current modal and opens the next one. The function also creates the const selectedBit finds the bit parameters and stores this as the var bitValue. This is then used to run a series of if statements that first confirm that the (selectedBit.rpm >= rpmValue) or else defaults to { rpmBit = selectedBit.rpm }.

The boolean material.soft and material.hard are then checked and the correct chip load value is then applied. This value is then stored as var hardness which is then used to calculate var feedRate via feedRate = (selectedBit.flutes * rpmBit * hardness). From there the var numPass is calculated by (Math.ceil(thicknessValue / (selectedBit.ced / selectedBit.flutes))). And then numPass is used to calculate the passDepth if the (selectedBit.ced <= thicknessValue) by (Math.round(((thicknessValue / numPass) + Number.EPSILON) * 100) / 100). Otherwise in the case where the selectedBit.ced is greater than the thickness value then a default of passDepth = thicknessValue is applied. These values are then all displayed to the user along with a few other parameter such as the angle of cut, the number of flutes, etc.

The user can exit the modal at any time by a closing it via the X or clicking outside the modal. This also resets the data stored in the application by trigger the refreshPage function.

FUTURE DEVELOPMENT

Continuing with this application in ITDEV-161 will be the development of filtering product by manufacture, manufacture tooling options, which in turn will filter by types of insert tools available (bits, blades, punches, plotting pens, crease tools, etc). A series of recommended parameters for cutting will then be presented. In addition the materials available will be greatly expanded on by having categories/types of materials which then have available materials displayed.

In addition the goal is for users to be able to enter their own materials and parameters to submit back to the application and stored by account. These can then be accessed by the individual user in the future and upon review by admin (developer) could be added to the general materials database. Which is the method I would like to move towards in the future which would probably entail converting quite a bit of this to php or at minimum using php to populate the JavaScript arrays.
